# Dreamine Software Ontology

Dreamine FullKit의 코드 지식 그래프를 재사용 가능한 의미 모델로 변환하는 도구입니다. `dreamine.schema.yaml`이 유일한 스키마 원본이며 JSON Schema, JSON-LD context, OWL, SHACL은 이 파일에서 생성합니다.

## 설계 원칙

- 코드 경로나 임시 그래프 ID와 분리된 안정 URI를 사용합니다.
- 한국어와 영어 레이블을 하나의 개념에 함께 저장합니다.
- 관계를 독립 리소스로 표현하여 근거와 신뢰도를 보존합니다.
- 일반 탐색용 관계는 직접 ObjectProperty로도 투영하고, provenance·근거·신뢰도가 필요한 관계는 `SemanticRelation` 재ification을 함께 유지합니다.
- 스키마(TBox)와 코드 인스턴스(ABox)를 분리합니다.
- C# 선언 타입은 Roslyn으로 원본 소스를 전수 확인하며, `implements` 관계나 이름 규칙으로 타입을 추정하지 않습니다.
- Understand Anything, Apache Jena/Fuseki, Neo4j n10s는 같은 JSON-LD/OWL/SHACL 산출물을 소비하는 어댑터입니다.

## 타입 정본과 Overlay

- 원본 Understand Anything 그래프는 ID·파일·구조 관계를 보존하는 raw graph입니다.
- 타입 레이블, 타입 필터, 아키텍처 검증, 챗봇 타입 답변은 `Roslyn source declaration → source-verified ontology Overlay → raw graph metadata` 순서로 신뢰합니다.
- 원본과 Overlay가 충돌하면 UI와 챗봇은 Overlay `rdf:type`을 사용하고 원본 타입은 provenance 및 진단 정보로만 노출합니다.
- Roslyn으로 확인할 수 없는 노드는 추정하지 않고 `unverified`로 남깁니다.
- 타입 보정과 경로 재매핑은 stable URI를 변경하지 않습니다. raw 타입과 raw 경로는 `raw_element_type`, `raw_source_path`에 진단 근거로 보존합니다.
- 원본 그래프에 없지만 Roslyn 선언으로 유일하게 식별되는 타입, 선언 전용 partial 메서드와 그 SourceFile은 `derived_source_verified` ABox 요소로 보강하며 `declaredIn`/`contains`를 함께 생성합니다.

## 경로와 생성 코드 정책

- `100. Library`에서 `200. Tests`로 이동한 테스트 소스와 프로젝트 파일은 동일 상대 경로의 실제 파일이 존재할 때만 source-verified 경로로 재매핑합니다. 근거가 없으면 `stale_quarantined`로 격리합니다.
- `*.Designer.cs`, `*.g.cs`, `*.g.i.cs`, `*.generated.cs` 및 `<auto-generated>` 소스는 선언 완전성·아키텍처 규칙·UI 기본 검색·챗봇 검색에서 제외합니다.
- 제외된 raw 노드는 삭제하지 않고 provenance 진단용으로 유지합니다. UI와 챗봇용 그래프에서는 `default_search_visible=false`인 노드를 소비하지 않습니다.

## Dreamine Event 의미 모델

- `*.xaml.Event.cs`의 실제 클래스와 `[DreamineEvent]` 필드로 연결된 구성 요소를 `DreamineEventComponent`로 표현합니다.
- Event 파일은 `EventComponentFile`, ViewModel과 구성 요소는 `hasEventComponent`, `[DreamineCommand("Event.X")]` 선언과 대상 동작 메서드는 `forwardsTo`로 연결합니다.
- `DreamineEventForwardTargetShape`은 forwarding 대상 메서드가 연결된 Event Component에 실제 존재하는지 검증합니다. 일반 C# event/handler 모델과는 별도 개념입니다.

## 생성

```powershell
.\Generate-DreamineOntology.ps1
```

생성 결과는 `.ua/ontology`에 저장되며 배포 시 `Dreamine.Web/wwwroot/understand/ontology`로 복사합니다. `.venv`와 생성 결과는 Git에서 제외하고 스키마, 생성기, 검증기만 관리합니다.

기본 생성은 Roslyn 전수 감사, 전체 LinkML 구조 SHACL, 전체 아키텍처 SHACL, Shape별 positive/negative fixture와 dotNetRDF 검증을 수행합니다. `source-audit.json`에는 선언 유형, ABox 타입 일치, 관계 무결성, partial 메서드 분류, dangling 관계, stable URI 충돌을 기록합니다. LinkML SHACL은 필수값·자료형·카디널리티를 검증하고, Overlay의 직접 ObjectProperty와 Dreamine 아키텍처 제약은 별도 architecture shapes가 검증합니다.

UI와 챗봇 소비자 그래프까지 다시 생성하려면 저장소 루트에서 다음을 실행합니다.

```powershell
powershell -ExecutionPolicy Bypass -File .\50_SETUP\Ontology\Generate-DreamineOntology.ps1 -RepositoryRoot $PWD
node .\50_SETUP\UnderstandAnything\Generate-ProjectGraphs.mjs $PWD
node .\50_SETUP\UnderstandAnything\Validate-ProjectGraphs.mjs $PWD
```

- 챗봇 정본: `.ua/knowledge-graph.json`
- UI 정본: `.ua/knowledge-graph.source-verified.ko.json`, `.ua/knowledge-graph.source-verified.en.json`
- raw 보존본: `.ua/knowledge-graph.ko.json`, `.ua/knowledge-graph.en.json`
- 우선순위 증거: `.ua/ontology/consumer-precedence.json`

## 확장 경계

- 개념과 속성 추가: `dreamine.schema.yaml`
- 기존 코드 그래프 매핑: `Build-OntologyGraph.mjs`
- 저장소 연동: JSON-LD를 Fuseki에 적재하거나 Neo4j n10s로 가져오는 별도 어댑터
- 품질 규칙: 생성된 SHACL에 프로젝트 정책용 shapes 파일을 추가
